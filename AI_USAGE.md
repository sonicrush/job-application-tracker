# AI Usage Documentation
## Tools Used
- Claude (From Co-Pilot integration in VSCode)

## Key Prompts
1. "Read schema.sql to understand the schema of the database. Then read README.md to understand the plan behind the application. Afterwards, look at every other respective file. Inside there are implementations of certain parts of the application. There are also comments of unfinished parts of the application.
Your task is to implement any missing parts of the application. Afterwards, review existing implementation and adjust for best practices. You may completely modify HTML files.".

## What Worked Well
- Including some implementation as guidance for AI generation
- Ensuring the AI follows application purpose by directing it to review the README.md file
- Asking it to explain certain parts of the implementation to better my understanding of what it did post key prompt.

# What I Modified
- Changed some field inputs to better match a user rather than a developer (It used to require JSON knowledge)
- Modified certain UI elements to improve UX

# Lessons Learned
- Always be detailed with the key prompt. AI is often best used when it is fed detailed descriptions of what needs to be done.
- AI is a very effective researcher. It will fulfil your request more closely to an expert than a database (Ex. VS using google search engine)
- AI will be an integral part of the industry. The tool is incredibly powerful. It will become critical to understand and use within work environments.
